# H5G Slot Math Rules

Apply these rules whenever work touches slot machine math, RTP, reel strips, weighted tables, random outcomes, Monte Carlo simulation, PAR sheets, jackpots, free spins, hold-and-spin, collectors, WYS/Loot Link/SF symbols, or certification math.

## Probability First

- Slot math must be probability-based.
- Do not create, tune, modify, or approve slot math from intuition, target feel, art direction, or simulation-only evidence when exact math is tractable.
- Define the game cycle, sample/state space, probability measure, payout random variable, EV, RTP, second moment, variance, standard deviation, hit frequency, win frequency, feature EV, and max-win probability.
- If required inputs are missing, stop and ask. Do not invent assumptions.

## Dependent Probabilities

- Default to dependent until proven independent.
- Do not multiply marginal probabilities for a joint event without an independence proof.
- Linearity of expectation may justify summing EV, but not hit frequency, win frequency, variance, max-win probability, or "any win" probability.
- Use full-screen enumeration, inclusion-exclusion, covariance/full distribution, dynamic programming, or Markov chains when events overlap or state persists.
- Apply caps per outcome/path before expectation.

## Simulation

- Monte Carlo is verification, not the model.
- Report seed, spin count, observed RTP, sample SD, standard error, confidence interval, event counts, and max observed win.
- Compare simulation to theoretical values.
- Block completion when simulation and theory do not reconcile.

## Completion Requirement

Final answers for math work must include probability basis, results, verification, and assumptions/blockers. If any required field cannot be completed, do not call the math done, approved, or inspection-ready.
