# Dependency Checklist

Default to dependent until proven independent.

## Stop And Model Jointly

Use joint enumeration, inclusion-exclusion, covariance/full distribution, dynamic programming, or Markov chains when any of these appear:

- Multiple paylines on one screen.
- Visible-anywhere scatters or coins.
- Visible positions on the same reel.
- Wilds, expanding reels, stacked symbols, substitutions, overlays, or multipliers.
- Retriggers, respins, hold-and-spin resets, persistent meters, collectors, or jackpots.
- Pick or wheel prizes without replacement.
- Max-win or prize caps.
- Dynamic gameplay, user choice, item loadout, class, power bet, ante, buy feature, or jurisdiction variant.

## Safe Uses Of Linearity

Linearity of expectation can justify summing EV components even when events are dependent.

It does not justify summing:

- Hit frequency.
- Win frequency.
- Variance or standard deviation.
- Max-win probability.
- "At least one" / union probabilities.
- Full payout distribution.

## Required Proof

For every multi-event metric, document one:

- Independence proof.
- Joint enumeration.
- Inclusion-exclusion terms.
- Covariance/full distribution.
- State model and transition probabilities.

If none is present, the math is incomplete.
